# Product

## Register

product

## Users

The primary users are a bootcamp instructor and around 10 medical faculty IT students with mixed programming backgrounds. Learners use the site during a 5-day, 20-hour hands-on class while following along in VS Code, GitHub, and the browser. Some learners only know programming basics, while others have practical experience, so the material must support different speeds without breaking the classroom flow.

## Product Purpose

This product is a course-book and slide surface for teaching HTML, CSS, TypeScript, Next.js, Tailwind, Supabase, deployment, authentication, roles, RLS, and LLM-safe coding through one guided IT issue reporting web application. Success means learners can read or watch the material step by step, follow the instructor during live teaching, and understand how frontend, backend, database, authentication, authorization, and security responsibilities fit together in a real web service workflow.

## Brand Personality

Practical, calm, followable. The voice should feel like an instructor who knows the real operational context and keeps learners moving with clear explanations, visible checkpoints, and enough detail to prevent confusion without overloading the room.

## Anti-references

The site should not feel like a marketing landing page, a decorative slide template, or a colorful showcase that competes with the lesson content. Avoid ornamental layouts, noisy effects, generic course-platform styling, and dense pages where learners cannot tell what to do next. The design should not hide core workflow diagrams, code, or checkpoints behind visual flourishes.

## Design Principles

1. Teach from the real workflow: every visual choice should help connect HTML, CSS, TypeScript, CRUD, database, auth, roles, RLS, deploy, and security into one understandable system.
2. Keep the class followable: prioritize readable Thai prose, clear slide framing, predictable navigation, and visible checkpoints so learners can recover if they fall behind.
3. Practice what the course teaches: the site should model the same clear structure, maintainable components, and safe implementation habits that learners are expected to adopt.
4. Use slides as a teaching lens, not a second content source: slide mode should present the same MDX material in a live-teaching format so the instructor does not maintain duplicate content.
5. Keep visual energy quiet and useful: VS Code Dark Plus, code-first surfaces, diagrams, and restrained controls should make long sessions comfortable.

## Accessibility & Inclusion

Target WCAG AA contrast for lesson text, navigation, controls, diagrams, tables, and code blocks. Respect reduced-motion preferences and avoid motion that distracts from instruction. The interface should remain readable on projector-sized desktop screens and usable on smaller screens for review. Thai text should keep comfortable line length, spacing, and hierarchy so learners can scan while coding.
