## Slide 3: Tailwind CSS คืออะไร

### Tailwind CSS

Tailwind คือ utility-first CSS framework

แทนที่จะเขียน CSS แบบนี้:

```css
.panel {
  background: white;
  border-radius: 8px;
  padding: 24px;
}
```

เราเขียน class ใน JSX/TSX แบบนี้:

```tsx
<section className="rounded-lg border border-slate-200 bg-white p-6">
  ...
</section>
```

Tailwind ไม่ได้ทำให้ไม่ต้องเข้าใจ CSS แต่ทำให้เราใช้ CSS concept ผ่าน class สำเร็จรูปได้เร็วขึ้น

## Slide 4: CSS ปกติ vs Tailwind

### CSS ปกติ

```tsx
<button className="primary-button">ส่งข้อมูล</button>
```

```css
.primary-button {
  border-radius: 6px;
  padding: 12px 16px;
  background: #0f766e;
  color: white;
}
```

### Tailwind

```tsx
<button className="rounded-md bg-teal-700 px-4 py-3 font-bold text-white hover:bg-teal-800">
  ส่งข้อมูล
</button>
```

### ข้อดี

- เห็น style อยู่ใกล้ component
- ลดการตั้งชื่อ class เอง
- ปรับ responsive และ state ได้เร็ว
- เหมาะกับ component-based UI

## Slide 5: ตรวจ Tailwind CSS ใน Next.js Project

### Next.js เวอร์ชันใหม่

ถ้าสร้าง project ด้วย `create-next-app` และเลือก recommended defaults ระบบจะเตรียม Tailwind CSS มาให้แล้ว

ดังนั้นขั้นแรกไม่ใช่ติดตั้งทันที แต่ให้ตรวจว่า project มี Tailwind อยู่แล้วหรือยัง

### สิ่งที่ควรเห็นใน project

```text
package.json
postcss.config.mjs
app/globals.css
app/layout.tsx
```

### ตรวจใน `package.json`

ควรเห็น package เหล่านี้:

```json
"tailwindcss": "...",
"@tailwindcss/postcss": "..."
```

### ตรวจใน `postcss.config.mjs`

ควรมี plugin ของ Tailwind:

```js
const config = {
  plugins: {
  "@tailwindcss/postcss": {},
  },
};

export default config;
```

### ตรวจใน `app/globals.css`

ควรมี `@import "tailwindcss";` อยู่ในไฟล์ global CSS

```css
@import "tailwindcss";
```

### ถ้ายังไม่มีจริง ๆ

ให้ติดตั้งเพิ่มที่ project root ซึ่งเป็น folder ที่มี `package.json`

```bash
npm install -D tailwindcss @tailwindcss/postcss
```

## Slide 11: Responsive Prefix

### หลักคิด

- class ที่ไม่มี prefix ทำงานกับทุกขนาดจอ
- เริ่มเขียนจาก mobile ก่อน
- `md:` เริ่มทำงานตั้งแต่จอกลางขึ้นไป
- `lg:` เริ่มทำงานตั้งแต่จอใหญ่ขึ้นไป

### ตัวอย่าง

```tsx
<div className="grid grid-cols-1 gap-4 md:grid-cols-2 lg:grid-cols-3">
  ...
</div>
```

### อ่าน class นี้

```text
grid-cols-1    -> mobile เป็น 1 column
md:grid-cols-2 -> จอกลางขึ้นไปเปลี่ยนเป็น 2 columns
lg:grid-cols-3 -> จอใหญ่ขึ้นไปเปลี่ยนเป็น 3 columns
```

### ใช้กับระบบเรา

- mobile: form field เรียงแนวตั้ง อ่านง่าย ไม่เบียดกัน
- tablet/desktop: ชื่อผู้แจ้งและอีเมลวางเป็น 2 คอลัมน์
- จอกว้าง: dashboard card วางเป็น 3 คอลัมน์ได้

## Slide 12: State Prefix

### หลักคิด

State prefix คือการบอกว่า class นี้จะทำงานเฉพาะตอน element อยู่ในสถานะนั้น

- `hover:` ทำงานเมื่อเอาเมาส์ไปชี้
- `focus:` ทำงานเมื่อผู้ใช้คลิกหรือ tab เข้ามาที่ input
- `disabled:` ทำงานเมื่อ element ถูกปิดการใช้งาน

### Hover

```tsx
<button className="bg-teal-700 hover:bg-teal-800">
  ส่งข้อมูล
</button>
```

### Focus

```tsx
<input className="focus:border-teal-700 focus:outline-none focus:ring-4 focus:ring-teal-100" />
```

### Disabled

```tsx
<button className="disabled:cursor-not-allowed disabled:bg-slate-300">
  ส่งข้อมูล
</button>
```

### อ่าน class นี้

```text
hover:bg-teal-800      -> hover แล้วปุ่มเข้มขึ้น
focus:ring-4           -> focus แล้วมีวงแหวนช่วยบอกตำแหน่ง
disabled:bg-slate-300  -> disabled แล้วสีจางลง
```

## Slide 18: ควรลบ CSS เดิมไหม

### คำตอบ

Day 3 คือช่วงย้าย UI จาก CSS เดิมมาเป็น Tailwind ดังนั้นย้ายส่วนไหนเสร็จ ให้ลบ CSS เดิมของส่วนนั้นออกจาก `app/globals.css` ทันที

### วิธีที่แนะนำ

1. ตรวจว่า Tailwind ถูกตั้งค่าไว้แล้วหรือยัง
2. แปลง navigation, page หรือ component ทีละส่วน
3. ลบ CSS เดิมของส่วนนั้นจาก `app/globals.css`
4. ตรวจหน้าเว็บ
5. เก็บเฉพาะ global style ที่จำเป็นจริง ๆ เช่น `@import "tailwindcss";`, `body`, CSS variables หรือ selector ที่ component อื่นยังใช้จริง

ตัวที่ควรระวังคือ selector กว้าง ๆ เช่น `main`, `section`, `button`, `table`, `input`, `textarea` เพราะมันอาจ override Tailwind class และทำให้ต้อง reset cache หรือไล่ debug ซ้ำโดยไม่จำเป็น

หลักคิดคือไม่ลบ CSS ทั้งไฟล์ในครั้งเดียว แต่ไม่ปล่อย CSS ของส่วนที่ย้ายเสร็จแล้วค้างไว้

### Checklist CSS เดิมที่ต้องไล่ลบระหว่างย้าย

ลบเมื่อส่วนนั้นเปลี่ยนเป็น Tailwind แล้ว:

- `header`
- `main`
- `footer`
- `nav`, `.app-nav`, `.app-nav-inner`
- `section`, `section + section`

## Slide 21: Common Mistakes

### ข้อผิดพลาดที่พบบ่อย

- คิดว่าต้องติดตั้ง Tailwind ใหม่เสมอ ทั้งที่โปรเจกต์ใหม่อาจมีมาให้แล้ว
- ลืมตรวจ `postcss.config.mjs` และ `@import "tailwindcss";`
- ลืม restart dev server หลังติดตั้ง package เพิ่มเอง
- สะกด class ผิด
- ใส่ class ยาวเกินไปจนอ่านยาก
- ลบ CSS เดิมเร็วเกินไปจนหน้าเว็บพัง
- ใช้สีเยอะเกินไปจน UI ไม่เป็นระบบ
- พยายามจำทุก class แทนที่จะเข้าใจ pattern

### Speaker Notes

Tailwind ควรช่วยให้ทำงานเร็วขึ้น ไม่ใช่ทำให้ผู้เรียนกังวลว่าต้องจำ class ทั้งหมด

## Slide 22: Recap ชั่วโมงแรกของ Day 3

### สิ่งที่ได้เรียน

- Tailwind คือ utility-first CSS framework
- Tailwind ใช้ CSS concept เดิม เช่น spacing, color, border, layout, state
- Next.js project ที่ไม่ได้เลือก Tailwind ตั้งแต่แรกสามารถติดตั้งเพิ่มได้
- `globals.css` ยังใช้ import Tailwind และเก็บ global style ที่จำเป็นได้
- การแปลง CSS เดิมไป Tailwind ควรทำทีละส่วน และลบ CSS เดิมของส่วนนั้นทันทีหลังย้ายเสร็จ

### ต่อไป

เราจะปรับ component จาก Day 2 เช่น `IssueForm`, `IssueList` และ `StatusBadge` ให้ใช้ Tailwind และ responsive layout ที่พร้อมใช้งานมากขึ้น

## คำศัพท์สำคัญ

| คำศัพท์ | ความหมาย |
|---|---|
| Tailwind CSS | CSS framework แบบ utility-first |
| Utility class | class สำเร็จรูปที่แทน CSS property |
| Responsive prefix | prefix เช่น `md:` หรือ `lg:` สำหรับ breakpoint |
| State prefix | prefix เช่น `hover:`, `focus:`, `disabled:` |
| `globals.css` | CSS กลางของ Next.js app |
| PostCSS | เครื่องมือประมวลผล CSS ที่ Tailwind ใช้ร่วมกับ Next.js |
