## Course Content Workflow

งานในโปรเจกต์นี้เป็นงานจัดเตรียมเนื้อหาสำหรับการสอน

- เมื่อผู้ใช้ขอแก้ slide หรือเนื้อหาบทเรียน ให้แก้ไฟล์โดยตรง
- ไม่ต้องสร้าง design spec, implementation plan หรือเอกสารประกอบ
- ไม่ต้องใช้ brainstorming หรือ Superpowers workflow เว้นแต่ผู้ใช้ร้องขอโดยตรง
- ถ้าคำขอชัดเจน ไม่ต้องถามยืนยันซ้ำ
- รักษาภาษาให้อ่านง่ายและเหมาะกับนักศึกษาระดับเริ่มต้น
- ตรวจ content และ build หลังแก้ตามความเหมาะสม
- ห้าม commit หรือ push จนกว่าผู้ใช้จะสั่ง
